Direções para produção web design mais clara e controlada

Saiba como você quer a sua pagina, e passe para o desenvolvedor.

Passar toda informar necessária pro elemento, se algo não esta claro deixe por escrito os detalhes através de comentários e/ou ilustrações como o elemento deve se comportar.

Exemplificar elementos que se repetem muito separados do layout da página, dessa maneira as pequenas variações acidentais de posicionamento interferem no desenvolvimento.

Informar limites de tamanho, proporções se a medida é estática ou percentual. Caso o elemento seja muito complexo ou voce não tenha certeza de como tamanho deve se comportar exemplificar o mesmo elemento em 2 tamanhos de tela (diferentes mas dentro do mesmo layout de desktop ou mobile), deve mostrar melhor para o desenvolvedor como o elemento funciona.

Exemplificar o elemento com muito e mínimo conteudo. E se necessário com elementos vasios (dificilmente um elemento não tem titulo, mas a falta de texto introdutorio e imagens é comum).

Exemplificar os diferentes estados do elemento (veja mais em elementos mutaveis). Exemplificação na propria pagina, tabelas com todos os estados ou notas como o elemento precisa ser detalhado depende da complexidade do mesmo.

Informar qual elemento está relacionado a propriedade (margem é relativa ao elemento pai, ao ultimo elemento interno ou ao elemento posterior).

Devido a variação de tamanho de tela a variação de largura é inevitável então é importante verificar como o mesmo elemento vai funcionar numa tela menor(dentro das predefinições de mobile e desktop).

Elementos que tem uma altura proporcional a largura são especialmente problemáticos com isso, podendo ser grandes ou pequenos demais dependendo da tela. Teste para saber se há necessidade de uma delimitação de tamanho máximo ou mínimo.

O editor pode acabar colocando muito mais ou muito menos caracteres que o esperado, Por algum erro ou alguma informação, elemento ou imagem pode não ser devidamente carregado.

Por isso é sempre importante definir limites para conteúdo em excesso  ou comportamentos que previnam a falta de conteúdo. Isso também vale para conjunto de elementos, se uma linha tem 3 cards o alinhamento continua o mesmo se for adicionado só dois? Um carrossel com uma imagem se manterá um carrossel?

Elementos que ativam uma animação, transição e mudança. Essas informações são passadas melhor de maneira escrita (nota ou tabela com situações) ou com um exemplo muito detalhado (o elemento em funcionamento ou um vídeo com as situações).

  • Oque ativa
  • se é infinito ou limitado
  • como vai funcionar
  • se possível um exemplo

Ex.: O quadrado flutuante a animação é ativada no hover do elemento total, o quadrado deve girar 90º uma vez e voltar ao estado inicial após o hover.

Hover

Os elementos clicáveis ainda são entendíveis sem o hover.

Existem telas de tamanho intermediário, logo o mesmo tamanho de tela pode ser um notebook ou um tablet, Informações que são exibidas em hover podem estar sempre visiveis em em touch ou ter outro comportamento (o desenvolvedor pode resolver especificar isso facilmente)

Focus

Não peça para remover o focus dos elementos (regularmente um outline em volta do inputs), eles são de extrema importância.

Você pode estilizar o focus mas isso fará dele visível após click. Você pode só manter o focus padrão, ele é uma ferramenta prática e só será visto quando necessário.

Porcentagem

Funciona a partir no tamanho do elemento pai (em caso de posicionamento absoluto o elemento q ele está relacionado e fixo se considera a tela).

VW e VH

Porcentagem da tela. 1vh 1% da altura da tela e 1vw 1% da largura da tela.

Pixel

Unidade mais comum para elementos estáticos.

Evitar o uso de 0.5px, o navegador não mostra meio pixel e arredonda (para baixo). E pixels quebrados causam desalinhamentos com elementos internos podendo ou não deixar um pixel vazio entre o elemento e conteúdo.

Rem

Unidade mais comum para fontes, é o tamanho padrão de fonte do navegador e pode ser configurado no navegador (muito utilizado por pessoas com dificuldade de visão).

Normalmente equivalente a 16px, mas pode ser configurado pra outro valor para facilitar estilização do site. Desta maneira você pode aumentar ou reduzir todas as fontes do site e manter a mesma hierarquia.

O tamanho fonte base desse documento é 1rem (equivalente a 16px).

Em

É equivalente ao tamanho da fonte do elemento(ou do pai). Utilizada para medidas que precisam ser relacionadas ao tamanho de fonte e serão utilizadas em elementos diferentes com diversos tamanhos de fonte.

O grid pode funcionar de maneiras diferentes, é importante caso use um grid fora do comum conversar com um desenvolvedor sobre as possibilidades e praticidade.

O grid é o elemento mais importante para responsividade, ele tem que ser pensado para todos os tamanhos de tela. Definir Break points e pensar tamanhos intermediários os tamanhos intermediários não precisam ser exemplificados, mas precisam ser considerados na definição do grid.

A “direção” do tamanho, que foi definido para o grid para um tamanho de tela, essas definições servem para esse tamanho para que intervalo de tamanho de tela. Basicamente um tamanho pode ser pra telas igual ou acima daquele tamanho ou telas até aquele tamanho.

Regularmente o grid é composto

  • Número de colunas
  • Tamanho Máximo do container
  • Calha
  • Margem min externa

O tamanho das colunas é normalmente definido por porcentagem(o número de colunas é o suficiente). Pode ser mantido um tamanho fixo mas isso demanda um Container sempre menor  que a tela (oque não funcionará bem mobile).

É importante seguir o Grid no site principalmente na contenção dos elementos, Você pode fazer elementos fora do grid mas esses demandam mais tempo e trabalho além de serem mais difíceis de serem adaptados nos tamanhos de tela intermediários.

Definir break points, para intervalos de tela.

Se o tamanho máximo do container for menor que o tamanho mínimo do intervalo - margem mínima o container terá 100% da largura logo variará mais.

Container 100% ou menor que tela?

O container  menor que a tela dá mais controle ao designer pra elementos que podem ficar grandes ou apertados demais já que há uma grande variação de largura, mas em telas pequenas (desktop ou mobile) a tela 100% proporciona mais espaço.

Pode ser um tamanho fixo em pixels, uma porcentagem ou até um porcentagem da tela, normalmente é usado em pixels.

Normalmente no código ela é formada de metade da calha em cada coluna, logo elementos que contam com a calha são complicados de serem implementados.

 esse atributo é mais necessário em casos onde o container ocupa 100% da tela, para definir a distância mínima entre conteúdo e borda, também pode ser usada para páginas onde o conteúdo tem um fundo de cor diferente do fundo do site

Quando Utilizar um elemento que não terá o posicionamento padrão é importante informar qual o elemento que ele está relacionado e as coordenadas a serem seguidas, As coordenadas podem ser em qualquer unidade, e em relação  ao topo, final,  esquerda e direita.

As coordenadas são as mesmas independente do tamanho do elemento ou tela, por isso é importante informar se ela é relativa ao topo ou fundo.

Estática

Posicionamento padrão dos elementos, no qual o elemento “ocupa um espaço” na tela, influência e é influenciado  pelos elementos antes e depois.

Absoluta

Relativo a um outro elemento (que precisa conter esse elemento). a posição é fixa em relação a esse elemento é definida por coordenadas.

Se você rolar a tela o elemento vai ficar preso ao posicionamento do elemento.

Fixa

Relativo a tela, a posição é fixa em relação a tela e é definida por coordenadas.

Se você rolar a tela o elemento continuará na mesma posição.

Caso o elemento seja relativo ao final da página ou uma porcentagem da altura é importante informar o valor da coordenada ou o tamanho da tela utilizada, Uma alternativa para alinhamento ao final é já colocar o elemento no final do mockup. (Plataformas como o figma tem o atributo fixed para o elemento, só precisa ser atribuida a direção correta)

Pegajoso Sticky

É uma posição que varia entre o comportamento padrão(estático) e o fixo. O elemento não ultrapassa o elemento pai, mas enquanto dentro dele, ele fica fixo.

Apesar de ser um comportamento já com um atributo de CSS muitas vezes este não pode ser utilizado, oque faz ser necessário a utilização de código extra (Javascript).

Blocos com diferentes elementos internos

Por o editor não adicionar ou até um bug um elemento pode estar vazio. É sempre bom prever esses casos.

Só remover o elemento, com margem.

Manter o espaço relativo a altura e largura do elemento.

Ter um espaçamento diferente entre o elemento anterior e o próximo (normalmente o mesmo que uma altura mínima)

Um elemento para quando estiver vazio (imagem place holder ou texto genérico)

Considerar se for adicionado uma imagem ou texto maior ou menor que o esperado, como os outros elementos se comportam em relação a essa diferença de tamanho.

  • Altura mínima e máxima
  • Limite de linhas (veja na parte de texto)
  • Alinhamento dos elementos nessas variações
  • Tamanho dos blocos laterais

Caso o elemento tenha uma altura maior que os elementos internos, os elementos internos podem ter um alinhamento pré definido

  • Alinhar ao topo ou final, a esquerda ou direita
  • Alinhar ao centro (vertical ou horizontal)
  • espaçar igualmente os elementos
  • um único elemento alinhado ao final

Elementos como carrosséis que mudam de conteúdo sem o recarregamento da página, “pedem” uma altura fixa pois a troca de conteúdo de tamanhos diferentes resulta na movimentação vertical dos elementos posteriores.

Isso implica no limite dos elementos internos, e como funcionará o alinhamento em conteúdos menores que o regular.

Exemplo: Um carrossel que contém título, parágrafo e botão (nessa ordem). Se for limitado somente a altura  geral um texto grande resultará na perda do botão, Logo deverá ser definido uma altura para o título e parágrafo e se for o caso sempre ser visível parte parágrafo um limite de linhas para o título e para o parágrafo.

Sempre definir comportamentos para situações de limite, você pode exemplificar a situação com um grande texto e texto nenhum ou deixar um comentário com as especificações.

Sugestão dar preferencia a textos grandes no mockup, além de te servir como alerta em como o elemento vai se comportar, muitas vezes o texto máximo já oferece informação suficiente pro desenvolvedor.

O editor do site pode colocar um texto grande demais eu texto nenhum, e acontece com frequencia.

Evitar usar sempre o tamanho ideal. (mesmo que fique mais bonito no mockup)

Usar uma caixa delimitadora com um conteúdo maior que o mostrado isso ilustra que o conteúdo deve ser cortado e que o elemento tem um altura máxima que deve ser respeitada. Isso também funciona no caso contrário é o elemento tem uma altura mínima.

Usar um texto cortado e elipses indicando como proceder em casos que ultrapassem o limite de linhas.

Informar por escrito as especificações necessárias, funciona bem e poupa tempo de produção dos exemplos. (Mas a produção de exemplos de mínimo e máximo é o ideal  se quer visualizar o resultado)

Deixar o alinhamento automático da caixa de texto (terminando na altura na linha), dessa maneira o espaçamento entre elementos vai estar correto.

O alinhamento errado pode causar diferenças na versão final do site (se o desenvolvedor não se atentar ao erro do mockup) ou gasto de tempo do desenvolvedor que vai ter calcular o tamanho correto.

Quando em texto deixar o espaço entre parágrafos visível, não há necessidade de fazer em todo o texto mas pelo menos um exemplo principalmente se o espaço for importante.

Sobre o espaço entre parágrafos, pode ser definido o espaço entre parágrafos e paralelo a isso o texto pode usar uma linha vazia como espaçamento é importante considerar se você quer esse espaçamento em todos os parágrafos e também como será o texto recebido pelo elemento(principalmente em um site já existente). Se o texto recebido tem linhas vazias para espaçar parágrafos e margem inferior o resultado será espaço dobrado, o contrário (sem linhas vazias e sem margem) resultará em nenhum espaçamento entre parágrafos.

Definir todos os elementos que poderão ser utilizados na área, exemplos:

  • Parágrafo
  • Títulos (diferentes níveis, quantos puderem ser adicionados)
  • Link
  • Lista com marcadores ou enumerada
  • Imagens (podendo conter descrição)

Menos comuns:

  • Tabela
  • Citação(Blockquote)
  • Botão entre textos
  • Caixas de texto
  • Elementos específicos site

Todos os elementos devem ter uma margem  (não inclui links) definida normalmente a margem  inferior , assim o espaço entre parágrafo seguido de uma lista será definido pelo parágrafo, mas também pode ser feito a partir da superior (se for coerente para o seu layout, mas mais incomum).

Alguns elementos podem ter margem superior (títulos normalmente pedem um espaçamento superior maior que o padrão do texto), caso um elemento no texto tenha uma margem inferior seja seguido por um de margem superior prevalecerá a maior margem (exemplo um parágrafo com margem inferior 20 seguido de um titulo com margem superior 40, terá um espaçamento de 40 entre os elementos). Atenção esse tipo de margem funciona bem dentro do texto mas não em elementos complexos.

Logo depois de definir todas as margens inferiores, você também pode atribuir uma margem superior para elementos que necessitam de um espaçamento maior (titulos, imagens, citações).

Você também pode definir espaçamentos específicos entre 2 elementos (parágrafo seguido lista, título seguido de imagem), mas é ideal definir a margem padrão para evitar que algo fique com o espaçamento errado.

As escolhas vão se aplicar a todas as imagens então não posicione a imagem na máscara livremente.
